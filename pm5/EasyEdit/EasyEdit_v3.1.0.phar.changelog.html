Compatibility with API 5.12.0

Feature Changes:

  • Larger tasks now render a lot faster on the client
  • Faster response times for small tasks (e.g. brush execution)
  • Added support for reading McStructure format
  • //merge no longer replaces foliage blocks with air
  • Added a //desel command to remove the current selection
  • Block parser no longer implicitly ignores broken state data
  • Improved block parser error messages
  • Added support for enchanting tables and ender chests in schematics
  • terrain ignored block setting now allows to use block tags

Bug Fixes:

  • Fixed fast-set crashing on api 5.12+ ~matsuyuki
  • Fixed rotating and flipping ~Matze
  • Fixed custom blocks not being displayed correctly after editing
  • Fixed tool cooldown allowing to interact with the world
  • Fixed McEdit schematics missing their top 64 blocks
  • Fixed schematics pasting one extra layer of air into each direction
  • Fixed schematics with detailed tile data crashing on load
  • Fixed schematics with a high amount of tile entities running into open file limits
  • Fixed containers in schematics failing to load their contents
  • Fixed enchantments not being properly converted
  • Fixed furnaces in schematics failing to load their contents
  • Fixed pathfinding max setting not working correctly
  • Fixed first task after server start not properly loading height map
  • Fixed replace command crashing when not specifying any arguments
  • Fixed setting block tags (e.g. //set #glass) setting update blocks
  • Fixed //fill and //drain tasks crashing when reaching world limitations
  • Fixed benchmarks saving their undo data

Internal Changes:

  • Tasks can now run on the main thread directly
    • This makes small edits a lot faster as no thread switch is needed
    • The thread will be chosen dynamically based on the size of the task
  • Selections from copy / undoing are now properly passed between threads
  • Large internal cleanups to chunk splitting
